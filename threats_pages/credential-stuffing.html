Credential Stuffing

Description: Attackers use stolen username/password combinations from one breach to attempt to log into other accounts.

Impact: Account takeovers, financial fraud.

Examples: LinkedIn, Dropbox credential stuffing incidents.

Credential stuffing is a type of cyberattack where attackers use stolen username-password pairs, typically obtained from data breaches, to gain unauthorized access to user accounts on other websites. Since many users reuse passwords across multiple sites, attackers try these stolen credentials across various platforms in an attempt to gain access. Protecting against credential stuffing requires both strong user authentication practices and backend defenses.

Here are the key strategies to protect against credential stuffing attacks:

  1. Use Unique, Strong Passwords
    • Encourage users to use unique passwords for every account they create. If a password is compromised on one platform, it cannot be reused by attackers on other platforms.
    • Promote the use of strong passwords (at least 12 characters, combining uppercase, lowercase, numbers, and special characters) to make brute-force attacks less effective.
    • Consider implementing password complexity rules, but also make sure they are not so complicated that users resort to weak patterns (e.g., "Password123!").
  2. Implement Multi-Factor Authentication (MFA)
    • Multi-factor authentication (MFA) is one of the most effective ways to defend against credential stuffing. Even if attackers have valid credentials, they will be unable to access the account without the second factor (such as a one-time code sent to a phone, biometrics, or hardware token).
    • Enforce MFA across all sensitive systems, including administrative or high-value user accounts.
  3. Monitor and Limit Login Attempts
    • Implement rate limiting on login attempts to slow down credential stuffing attacks. By limiting the number of login attempts a user can make in a short period, you can reduce the attacker's ability to test multiple credentials quickly.
    • Use progressive delays (throttling) after failed login attempts, increasing the delay with each successive failure.
    • Temporarily block IP addresses or users after a predefined number of failed login attempts (lockout mechanisms), but ensure there are safeguards to avoid locking out legitimate users.
  4. Use CAPTCHA or reCAPTCHA
    • Use CAPTCHA or reCAPTCHA to ensure that login attempts are made by real users and not automated bots. CAPTCHA can be triggered after a certain number of failed login attempts or suspicious behavior.
    • CAPTCHA adds a challenge that makes it harder for attackers to use automated tools to test stolen credentials, slowing down or preventing credential stuffing attacks.
  5. Leverage Passwordless Authentication
    • Consider implementing passwordless authentication methods such as biometrics, magic links, or one-time passcodes. By eliminating passwords altogether, you remove the main vector for credential stuffing.
    • FIDO2/WebAuthn standards support passwordless authentication using hardware security keys or biometric authentication, providing a more secure and user-friendly login experience.
  6. Enable Account Anomaly Detection
    • Implement systems to detect unusual login activity such as multiple failed login attempts, logins from unfamiliar locations or devices, or rapid login attempts from different IP addresses.
    • Use behavioral analytics to detect anomalies in user behavior that may indicate a credential stuffing attack. For example, if a user suddenly logs in from multiple geographic locations or a large number of accounts are accessed from the same IP, it could indicate an ongoing attack.
  7. Deploy Credential Monitoring Services
    • Use credential monitoring services (also known as breach detection services) that alert users and administrators when stolen or leaked credentials appear in public breach repositories or dark web markets.
    • Inform users when their credentials have been found in a breach and prompt them to change their passwords immediately.
  8. Use IP Blacklisting and Geo-Blocking
    • Maintain an IP blacklist of known malicious IP addresses or those commonly associated with credential stuffing attacks. Regularly update this list and block login attempts from those IPs.
